# API Reference

The OMPAY Payment Gateway API is organized around REST. All endpoints return JSON and use standard HTTP response codes.

## Base URLs

| Environment | Base URL |
|-------------|----------|
| UAT or Sandbox | `https://api.uat.gateway.ompay.com` |
| Production | `https://api.gateway.ompay.com` |

## Authentication

All endpoints require authentication. See the [Authentication](/docs/getting-started/authentication) guide for details on obtaining credentials and generating signatures.

## Endpoint Groups

### Bank Hosted

Endpoints for redirect-based checkout flows where OMPAY handles the payment page.

- Create Checkout Order
- Check Payment Status

### Merchant Hosted

Endpoints for server-to-server integrations with encrypted card data and full checkout control.

- Create Order
- Initiate Transaction
- Get Transaction Status
- Initiate Refund
- List Digital Cards
- Get Digital Card
- Delete Digital Card
